UNION AFRICAINE — AUCTC

CAERT

Formation utilisateurs & administrateurs

Plateforme de reporting d'incidents sécuritaires · v2.0.0-prod

Classification : interne — sensible

Objectifs

  • Comprendre le rôle de CAERT dans la chaîne AUCTC
  • Connexion, navigation, langue FR/EN
  • Saisie et import Excel d'incidents
  • Workflow validation → publication
  • Analyses, synthèse direction, carte SIG

Profils utilisateurs

  • Point focal — saisie périmètre pays
  • Analyste — consultation et reporting
  • Admin — validation et synthèse
  • Super-admin — comptes et référentiels

Statuts incident

  • En attente — soumis, non validé
  • Publié — visible KPIs, carte, analyses
  • Rejeté — motif obligatoire, correction possible

Seuls les incidents publiés alimentent les indicateurs officiels.

Workflow contributeur

  • Saisir ou importer → statut En attente
  • Admin publie ou rejette
  • Rejet : lire motif → corriger → resoumettre

Import Excel

  • Format .xlsx — colonne A = date d'attaque
  • Libellés référentiels identiques à la base
  • Rapport succès / partiel / échec
  • Tous les imports restent En attente

Validation (admin)

  • File de validation — incidents en attente
  • Vérifier cohérence avant publication
  • Motif de rejet clair et actionnable
  • Audit automatique de chaque action

Synthèse direction

  • KPIs : publiés, en attente, rejetés, décès
  • Top 10 pays (pas exhaustive)
  • Accès ROLE_ADMIN

Analyses & Carte

  • Graphiques tendances — données publiées
  • Comparatif période / région / indicateur
  • Carte SIG — agrégation au centroïde pays

Sécurité

  • Données sensibles — usage institutionnel
  • Pas de partage d'identifiants
  • Déconnexion sur poste partagé
  • « Rester connecté » — poste personnel seulement
